# 자가 개선 루프 워크플로

> nestjs-tenant-shield의 하네스 엔지니어링 구조를 그대로 이식한다.
> 매 작업마다 평가를 누적하고, 누적된 데이터를 Opus가 회고해서
> [CLAUDE.md](./CLAUDE.md)를 다듬는 구조.
>
> 이 파일은 **하네스 운영 매뉴얼**이다.

---

## 0. 한 줄 요약

```
작업 → /eval → ccusage 차분 → evaluations.md 누적 → /eval-review (20건) → CLAUDE.md 수정 → 반복
```

---


## 2. 슬래시 커맨드 셋업

### 2.1 `/eval` 설치

`~/.claude/commands/eval.md`:

````markdown
---
description: 작업 평가 + 로그 파일에 저장
---

방금 수행한 작업을 아래 양식으로 평가하고,
평가 결과를 `~/.claude/evaluations.md` 파일에 append 해줘.

파일이 없으면 생성. 각 평가 위에 헤더 `## YYYY-MM-DD HH:MM` 추가.

---

**작업 요약**: (한 줄)
**난이도**: 상 / 중 / 하
**사용 모델**: (Sonnet / Opus / Haiku 조합)
**불확실 지점**: (검증 못 한 것. 없으면 "없음")
**모델 적정성**: 적정 / 오버 / 언더
**반복 패턴 후보**: (공통으로 통할 만한 규칙 1개. 없으면 "없음")
````

### 2.2 `/eval-review` 설치

`~/.claude/commands/eval-review.md`:

````markdown
---
description: 누적된 평가 로그를 Opus가 회고하고 개선안 제안
---

`~/.claude/evaluations.md` 파일을 읽고:
1. 평가 건수, 기간, 난이도/모델 적정성 분포
2. 불확실 지점 TOP 3
3. 4건 이상 반복 패턴 추출
4. CLAUDE.md / critical-notes.md 개선 제안 (근거 명시)
5. 3줄 요약

원칙: 1~2건짜리는 "관찰만" 분류. 즉시 반영 금지.
````

---

## 3. 회고 결과 → 문서 반영 규칙

| 회고 결과 유형 | 반영 파일 |
|---|---|
| 운영 규칙, 모델 선택, 컨벤션 | [CLAUDE.md](./CLAUDE.md) |
| 실제 발견된 사고/버그 패턴 | [critical-notes.md](./critical-notes.md) |
| 다음 작업 진입점·체크포인트 | [process.md](./process.md) |
| 슬래시 커맨드 추가/수정 | `~/.claude/commands/` + 이 파일 §2 |

4건 이상 반복 패턴만 CLAUDE.md에 격상. 1~2건은 process.md §10 "관찰만" 메모.
